# Сервис русско-украинского морфологического анализа (russian-bg)

> Этот проект — серверный модуль морфологического анализа для **приложения «鹅语菌»**,  
> предоставляющий склонение, спряжение, морфологический разбор, проверку орфографии и другие функции.  
> Источник и клиентское приложение: **鹅语菌 App (страница загрузки: `https://egg404.com`)**.

---

## 🧩 Обзор проекта

- **Стек технологий**:
  - Backend-фреймворк: FastAPI (Python)
  - Морфологический движок: `pymorphy2` (поддержка русского `ru` и украинского `uk`)
  - HTTP‑сервер: uvicorn
  - Статический фронтенд: чистый HTML + CSS + JS (каталог `static/`)
- **Сценарии использования**:
  - Обучающие сервисы по русскому и украинскому языкам
  - Тренажёры по формам слов и грамматике
  - Языковой модуль для мобильного приложения «鹅语菌»

## 📁 Структура проекта

- `app/main.py` — точка входа FastAPI, регистрация маршрутов, CORS, middleware и статики.
- `app/controllers/morph_controller.py` — контроллер HTTP‑API морфологического анализа.
- `app/services/morph_service.py` — основной сервис морфологии, инкапсулирующий вызовы `pymorphy2`.
- `app/models/schemas.py` — Pydantic‑схемы запросов и ответов.
- `static/` — статические файлы (HTML, CSS, JS).
- `docs/frontend_integration.md` — руководство по интеграции фронтенда.
- `DEPLOYMENT.md` — пример деплоя через панель BaoTa (宝塔面板).
- `requirements.txt` — список Python‑зависимостей.
- `start.sh` — вспомогательный скрипт запуска (опционально).

## 🚀 Локальный запуск

### 1. Подготовка окружения

1. Установите Python (рекомендуется **3.9** или **3.11**).
2. Клонируйте репозиторий:

```bash
git clone https://github.com/3441152376/russian-bg.git
cd russian-bg
```

3. Создайте и активируйте виртуальное окружение:

```bash
python3 -m venv .venv
source .venv/bin/activate  # В Windows: .venv\Scripts\activate
```

4. Установите зависимости:

```bash
pip install -r requirements.txt
```

### 2. Запуск сервиса

Запуск через uvicorn:

```bash
python -m uvicorn app.main:app --host 0.0.0.0 --port 5136
```

Или запуск встроенной точки входа:

```bash
python app/main.py
```

> Порт можно изменить через переменную окружения `PORT` или флаг `--port`.

### 3. Проверка работы

- Страница тестового фронтенда: `http://localhost:5136/static/index.html`
- Health‑check: `http://localhost:5136/healthz`
- Документация OpenAPI (Swagger): `http://localhost:5136/docs`

## 🔌 Обзор API

Все маршруты имеют префикс `/api/morph`. Основные эндпоинты (описаны в `app/controllers/morph_controller.py`):

- `GET /api/morph/smart-analysis`  
  - **Назначение**: «умный» анализ — определение части речи и вывод связанных словоформ.  
  - **Параметры**:
    - `word` — обязательный параметр, анализируемое слово.
    - `language` — опционально, `ru` или `uk` (по умолчанию `ru`).

- `GET /api/morph/pos-tagging` — морфологический разбор и разметка части речи.
- `GET /api/morph/declension` — полная парадигма склонения существительного (6 падежей × ед./мн. число).
- `GET /api/morph/adjective-declension` — склонение прилагательного.
- `GET /api/morph/verb-conjugation` — спряжение глагола по временам, лицам и числам.
- `GET /api/morph/stem-lexeme` — информация о стемах и лексеме.
- `GET /api/morph/spell-check` — проверка орфографии и предложения похожих слов.
- `GET /api/morph/normal-forms` — все возможные начальные формы.
- `POST /api/morph/text-analysis` — анализ текста (пословный разбор).
- `POST /api/morph/batch-analyze` — пакетный анализ списка слов.
- `POST /api/morph/generate-forms` — генерация словоформ по заданным граммемам.
- `POST /api/morph/grammar-check` — проверка грамматической согласованности слова с контекстом.
- `GET /api/morph/root-aspect-analysis` — анализ корня и вида глагола (совершенный/несовершенный).

> Подробные схемы запросов/ответов и примеры можно посмотреть напрямую в Swagger по адресу `http://localhost:5136/docs`.
